## 二、阶段详细任务

### 阶段 0：模拟器脚本（前置依赖，无代码依赖）

#### T001 — TCP 六通道帧模拟器

| 项 | 说明 |
|------|------|
| **文件** | `tools/mag_collector_simulator.py` |
| **功能** | 按物理帧格式封装 48 字节数据帧，1000Hz TCP 发送 |
| **关键参数** | `--host`, `--port`, `--rate 1000`, `--amplitude`, `--phases 0,30,60,90,120,150` |
| **帧生成** | 帧头 FM + 长度 48 + 递增序列 uint32 + 时间信息 + 6 通道 int24 数据 + 标志位 'X''Y''Z''x''y''z' + 校验和 + 帧尾 0D0A |
| **测试信号** | 每通道独立 50Hz 正弦波：`data[n] = int(A × sin(2π×50×n/1000 + φ) × 8388607/2.5/1000/40)` |
| **验证方式** | 用 Wireshark 或 `nc` 接收，检查帧头和帧尾，计数帧率是否 ~1000/s |

#### T002 — 串口 `$` 格式模拟器

| 项 | 说明 |
|------|------|
| **文件** | `tools/dollar_serial_simulator.py` |
| **功能** | 生成 `$xxxxxxx.xxxxxx\n` 格式 ASCII 数据，250Hz TCP 发送 |
| **关键参数** | `--host`, `--port`, `--rate 250`, `--baseline`, `--amplitude`, `--phase` |
| **数据生成** | `value = baseline + A × sin(2π×50×n/250 + φ)` → 格式化为 `$%07d.%06d` |
| **验证方式** | `nc` 接收，检查 `$` 开头和格式 |

> ⚠️ 阶段 0 完成后，可以独立验证两个模拟器的数据格式，不依赖 C++ 代码。

#### T004 — MagCollectorDecoder 状态机

| 项 | 说明 |
|------|------|
| **文件** | `include/MagCollectorDecoder.h` `src/protocol/MagCollectorDecoder.cpp` |
| **状态机** | 12 状态：WAIT_HEADER_H → WAIT_HEADER_L → WAIT_LENGTH_L → WAIT_LENGTH_H → WAIT_SEQUENCE(4B) → WAIT_TIMESTAMP(12B) → WAIT_CHANNELS(24B) → WAIT_CHECKSUM_L → WAIT_CHECKSUM_H → WAIT_FOOTER_L → WAIT_FOOTER_H → 校验 → emit/丢弃 |
| **粘包处理** | 状态跨 `feed()` 调用保持，帧尾匹配失败退回到 WAIT_HEADER_H 在已读数据中重新扫描 "FM" |
| **校验** | (1) checksum 验证 (2) 帧尾 0D0A 验证 (3) 通道标志位 ASCII 验证 |
| **信号** | `frameDecoded(MagCollectorFrame)` / `checksumError(QByteArray)` |

**状态机图示**：

```
                              ┌──────────────────────────────────┐
                              │          WAIT_HEADER_H            │
                              │  逐字节找 0x46 ('F')              │
                              └──────────────┬───────────────────┘
                                    找到 'F' │
                                             ▼
                              ┌──────────────────────────────────┐
                              │          WAIT_HEADER_L            │
                              │  必须是 0x4D ('M')，否则回到HEAD_H │
                              └──────────────┬───────────────────┘
                                    找到 'M' │
                                             ▼
                              ┌──────────────────────────────────┐
                              │    WAIT_LENGTH_L → WAIT_LENGTH_H  │
                              │  读 2 字节 uint16 LE 帧长         │
                              └──────────────┬───────────────────┘
                                             │
                                    ┌────────┴────────┐
                                    ▼                  ▼
                          WAIT_SEQUENCE(4B)    WAIT_TIMESTAMP(12B)
                                    │                  │
                                    └────────┬─────────┘
                                             ▼
                                    WAIT_CHANNELS(24B)
                              6×(int24数据 3B + 标志 1B)
                                             │
                                             ▼
                              WAIT_CHECKSUM_L → WAIT_CHECKSUM_H
                                             │
                                             ▼
                              WAIT_FOOTER_L(0x0D) → WAIT_FOOTER_H(0x0A)
                                             │
                                    校验通过? │
                                   ┌─────────┴─────────┐
                                   ▼                   ▼
                            emit frameDecoded    emit checksumError
                                   │                   │
                                   └─────────┬─────────┘
                                             ▼
                                      WAIT_HEADER_H
```

### 阶段 4：FFT 处理（依赖 T008）

#### T009 — KissFFT 集成 + FftProcessor

| 项 | 说明 |
|------|------|
| **文件** | `src/fft/kiss_fft.c` `src/fft/kiss_fft.h` `src/fft/kiss_fftr.c` `src/fft/kiss_fftr.h` `include/FftProcessor.h` `src/fft/FftProcessor.cpp` |
| **KissFFT 版本** | 从 [github.com/mborgerding/kissfft](https://github.com/mborgerding/kissfft) 取 `kiss_fft.c` `kiss_fft.h` `kiss_fftr.c` `kiss_fftr.h` |
| **FftProcessor 构造** | `FftProcessor(Config, DataBuffer*, extractor_lambda)` |
| **定时器** | `QTimer 20ms` 触发 `onTimer()` |
| **处理流程** | snapshot → 取 extractor 值 → 填 input[] → 乘 Hanning 窗 → `kiss_fftr()` → 取 bin=50 → `atan2` → 弧度转角度 → `emit phaseReady` |

**Hanning 窗**：

```cpp
for (int i = 0; i < N; ++i)
    window[i] = 0.5f * (1.0f - cosf(2.0f * M_PI * i / (N - 1)));
```

#### T010 — FftProcessor 单元测试

| 测试用例 | 数据 | 预期 |
|------|------|------|

| 已知初相 45° | `sin(2π×50×t + π/4)` | phase ≈ 45° ± 1° |
| 已知初相 -90° | `sin(2π×50×t - π/2)` | phase ≈ -90° ± 1° |
| 无信号（全零） | 1000 × 0.0 | phase 不 crash（静默处理） |
| 串口 N=250 | 同上，250 点 | phase ≈ 45° ± 1° |

## 五、风险与缓解

| 风险 | 概率 | 缓解措施 |
|------|:--:|------|
| 模拟器帧格式与实际设备不一致 | 中 | 帧结构通过头文件常量定义，改一处全局生效 |
| 1000Hz 下粘包频繁导致解码器状态机 bug | 低 | Mock 粘包场景在 T005 中覆盖，1000 帧压力测试 |
| KissFFT 非 2^n 长度性能不佳 | 低 | N=1000/250 都远小于 4096，即使 O(N²) 也 < 5ms |
| 相位差在 ±180° 附近跳变 | 中 | unwrap 作为 P2 可选功能，默认展示原始相位 |
| 双通道独立重连逻辑冲突 | 低 | 两套 ChannelManager 完全独立，无共享状态 |
