## 2. 帧解码：MagCollectorDecoder

### 2.1 帧格式

```
Offset  Size  内容
────────────────────────────────────
 0      2     帧头  0x46 0x4D  "FM"
 2      2     帧长  uint16 LE = 48
 4      4     序列  uint32 LE
 8      12    时间  WW/MM/DD/YY/HH/MM/SS/FF/MS(4B)
20      24    通道  6×(int24_LE 3B + 标志 1B)
44      2     校验  uint16 LE (offset 4~43 字节和)
46      2     帧尾  0x0D 0x0A  CRLF
────────────────────────────────────
总计: 48 字节
```

通道标志: `X`=0x58, `Y`=0x59, `Z`=0x5A, `x`=0x78, `y`=0x79, `z`=0x7A

### 2.2 状态机

```
WAIT_HEADER_H ─'F'→ WAIT_HEADER_L ─'M'→ WAIT_LENGTH_L → WAIT_LENGTH_H
    ↑                                        ↓
    │                                  WAIT_SEQUENCE(4B)
    │                                        ↓
    │                                  WAIT_TIMESTAMP(12B)
    │                                        ↓
    │                                  WAIT_CHANNELS(24B)
    │                                        ↓
    │                                  WAIT_CHECKSUM_L → WAIT_CHECKSUM_H
    │                                        ↓
    │                                  WAIT_FOOTER_L(0x0D) → WAIT_FOOTER_H(0x0A)
    │                                        │
    └──────────── 丢弃 ←── 失败 ──── [校验和 + 标志验证]
                                       │ 通过
                                       ▼
                                  emit frameDecoded()
```

**校验三项**：
1. 校验和 = 40 字节（序列+时间+通道）的字节累加，取 uint16
2. 帧尾 = 0x0D 0x0A
3. 通道标志 = `X Y Z x y z`

**int24 解码**（3 字节小端序 → int32 符号扩展）：

```cpp
int32_t decodeInt24(const uint8_t *bytes) {
    uint32_t raw = bytes[0] | (bytes[1] << 8) | (bytes[2] << 16);
    if (raw & 0x800000u) return static_cast<int32_t>(raw | 0xFF000000u);
    return static_cast<int32_t>(raw);
}
```

## 3. 串口解析：DollarParser

### 3.1 格式

```
$xxxxxxx.xxxxxx\n
 7位整数  6位小数

正则: ^\$(\d{7})\.(\d{6})$
```

### 3.2 解析逻辑

```
feed(bytes) → 追加到 m_pending
           → 按 \n 拆行
           → 每行: 匹配正则 → int + frac/1e6 → double
           → 非$行/格式错 → 跳过 + WARN
           → 半行(无\n) → 缓存到下次
```

## 4. 磁场计算：MagFormula

| 来源 | 公式 | 物理意义 |
|:--|:--|:--|
| **TCP ADC** | `Mag = raw × 100000.0 / 8388607.0` | 2.5V参考 / 23bit分辨率 × 1000 × 40 |
| **串口** | `Mag = value / 3.0` | 传感器标定系数 |

```cpp
namespace MagFormula {
    constexpr double ADC_MAX      = 8388607.0;   // 2^23 - 1
    constexpr double SCALE_FACTOR = 100000.0;    // 2.5 × 1000 × 40

    inline double fromAdc(int32_t raw) {
        return raw * SCALE_FACTOR / ADC_MAX;
    }
    inline double fromSerial(double value) {
        return value / 3.0;
    }
}
```

**关键**：TCP 只取 X1(data[0]) 和 Y1(data[1])，各自独立计算 Mag_X1、Mag_Y1，不合成矢量。

## 5. FFT 相位提取：FftProcessor

### 5.1 设计决策

| 参数 | TCP | 串口 | 理由 |
|:--|:--|:--|:--|
| 采样率 fs | 1000 Hz | 250 Hz | 设备固定 |
| FFT 点数 N | 1000 | 250 | **1Hz 分辨率**，50Hz bin 精确整数 |
| 频率分辨率 Δf | 1.0 Hz | 1.0 Hz | fs / N |
| 50Hz bin | 50 | 50 | N × 50 / fs |
| 窗函数 | Hanning | Hanning | 抑制频谱泄漏 |
| 输出率 | 50 Hz | 50 Hz | QTimer 20ms |

**为什么不用 N=1024？**
- 1024/1000Hz → Δf=0.977Hz → 50Hz 在 bin 51.2 → **频谱泄漏** → 相位不准
- 1000/1000Hz → Δf=1.0Hz → 50Hz 精确在 bin 50 → **无泄漏** ✅

### 5.2 滑动窗口 FFT 算法

```
每个 tick (20ms):
  1. snapshot() 从 MagDataBuffer 取最新 N 个 MagDataPoint
  2. 通过 extractor lambda 提取目标磁场值列
  3. 乘以 Hanning 窗: w[i] = 0.5 × (1 − cos(2π × i / (N−1)))
  4. kiss_fftr(samples, freqdata)  — 实数 FFT
  5. bin = 50
  6. φ = atan2(freqdata[bin].i, freqdata[bin].r) × 180/π
  7. 归一化到 [−180°, +180°]
  8. emit phaseReady(φ)
```

### 5.3 Hanning 窗

```cpp
for (int i = 0; i < N; ++i)
    window[i] = 0.5f * (1.0f - cosf(2.0f * M_PI * i / (N - 1)));
```

加窗目的：时域截断导致频谱泄漏 → Hanning 窗将旁瓣抑制到 −31dB，相位提取更精确。

### 5.4 FFT 库：KissFFT

- **类型**：实数 FFT (`kiss_fftr`)
- **输入**：N 个 float 实数
- **输出**：N/2+1 个复数（共轭对称）
- **许可**：BSD-3-Clause
- **性能**：N=1000 < 0.5ms（远低于 20ms 间隔要求）

```cpp
kiss_fftr_cfg cfg = kiss_fftr_alloc(N, 0, nullptr, nullptr);
kiss_fftr(cfg, input, output);
// output[bin].r → 实部, output[bin].i → 虚部
```

## 6. 相位差计算

### 6.1 计算公式

```
Δφ_X1 = normalize(φ_X1 − φ_Serial)    范围 [−180°, +180°]
Δφ_Y1 = normalize(φ_Y1 − φ_Serial)
```

### 6.2 归一化函数

```cpp
static double normAngle(double deg) {
    while (deg > 180.0) deg -= 360.0;
    while (deg < -180.0) deg += 360.0;
    return deg;
}
```

### 6.3 更新策略

三个 FftProcessor 异步产出相位值（50Hz 各产各的），UI 层维护最新缓存：

```
φ_X1 就绪 ─→ m_phiX1 = φ_X1 ─→ pushPhaseDisplay()
φ_Y1 就绪 ─→ m_phiY1 = φ_Y1 ─→ pushPhaseDisplay()
φ_Ser 就绪 → m_phiSer = φ_Ser → pushPhaseDisplay()
                                    │
                          相位波形:  [φ_X1, φ_Y1, φ_Serial]
                          差值波形:  [Δφ_X1, Δφ_Y1]
```

相位差只在 TCP 和串口都有值时才计算（`m_hasX1 && m_hasSerial`）。

## 7. 线程模型

```
┌────────────────┐  ┌────────────────┐  ┌────────────────┐  ┌────────────────┐
│ TCP通信线程     │  │ TCP解析线程     │  │ 串口通信线程    │  │ 串口解析线程    │
│ ChannelManager │  │ MagDecoder     │  │ ChannelManager │  │ DollarParser   │
│ TcpChannel     │→ │ → Mag_X1,Y1    │  │ SerialChannel  │→ │ → Mag_Ser      │
└────────────────┘  └───────┬────────┘  └────────────────┘  └───────┬────────┘
                            │                                       │
                     m_tcpMagBuffer                          m_serMagBuffer
                            │                                       │
                            └───────────┬───────────────────────────┘
                                        ▼
                              ┌────────────────┐
                              │   FFT 线程      │
                              │ FftProc_X1     │
                              │ FftProc_Y1     │
                              │ FftProc_Ser    │
                              │ → φ_X1,φ_Y1,φ_Ser │
                              └───────┬────────┘
                                      │ phaseReady (QueuedConnection)
                                      ▼
                              ┌────────────────┐
                              │   UI 线程       │
                              │ 相位差计算       │
                              │ RealTimeChart×2 │
                              └────────────────┘
```

**共 6 线程，全部 QueuedConnection，无锁竞争。**
