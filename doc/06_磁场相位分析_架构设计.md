# 06 — 磁场相位分析 · 架构设计

> **版本**：v1.0 | **日期**：2026-06-25

## 2. 数据流管道

### 2.1 TCP 数据流

```
┌─ Comm Thread (TCP) ─────────────────────────────────────┐
│  TcpChannel                                               │
│    ↓ readyRead(QByteArray)                                │
│  ChannelManager                                           │
│    ↓ readyRead                                            │
└──────────────────────────────────────────────────────────┘
                         │ Qt::QueuedConnection
                         ▼
┌─ Parse Thread (TCP) ─────────────────────────────────────┐
│  MagCollectorDecoder::feed(data)                          │
│    → 状态机解码 → 帧头"FM" + 帧长 + 序列 + 时间           │
│    → 6通道 int24 + 标志 → 校验和 + 帧尾 0D0A              │
│    ↓ 校验通过                                              │
│  emit frameDecoded(MagCollectorFrame)                     │
│    ↓                                                      │
│  MagFormula::fromAdc(frame.X1) → Mag_X1                   │
│  MagFormula::fromAdc(frame.Y1) → Mag_Y1                   │
│    ↓                                                      │
│  m_tcpMagBuffer.push({hostTimestamp, Mag_X1, Mag_Y1})     │
└──────────────────────────────────────────────────────────┘
                         │
                         ▼
┌─ FFT Thread ────────────────────────────────────────────┐
│  FftProcessor_X1 (consume m_tcpMagBuffer → X1列)          │
│    → snapshot最新1000点 → Hanning窗 → KissFFT              │
│    → bin=50 → atan2 → φ_X1                                │
│    ↓ emit phaseReady(φ_X1, ts)                            │
│                                                           │
│  FftProcessor_Y1 (consume m_tcpMagBuffer → Y1列)          │
│    → 同上 → φ_Y1                                          │
│    ↓ emit phaseReady(φ_Y1, ts)                            │
└──────────────────────────────────────────────────────────┘
                         │
                         ▼
┌─ UI Thread ──────────────────────────────────────────────┐
│  PhaseBuffer.push({ts, [φ_X1, φ_Y1]})                    │
│    → RealTimeChart #1 渲染相位波形                         │
│                                                           │
│  DiffBuffer.push({ts, [φ_X1-φ_Serial, φ_Y1-φ_Serial]})   │
│    → RealTimeChart #2 渲染相位差波形                       │
└──────────────────────────────────────────────────────────┘
```

### 2.2 串口数据流

```
┌─ Comm Thread (Serial) ───────────────────────────────────┐
│  SerialChannel                                             │
│    ↓ readyRead(QByteArray)                                 │
│  ChannelManager                                            │
│    ↓ readyRead                                             │
└──────────────────────────────────────────────────────────┘
                         │ Qt::QueuedConnection
                         ▼
┌─ Parse Thread (Serial) ──────────────────────────────────┐
│  DollarParser::feed(data)                                  │
│    → 按 \n 拆行 → 匹配 $(\d{7})\.(\d{6})                   │
│    ↓ 解析成功                                              │
│  MagFormula::fromSerial(value) → Mag_Serial                │
│    ↓                                                      │
│  m_serMagBuffer.push({hostTimestamp, Mag_Serial})          │
└──────────────────────────────────────────────────────────┘
                         │
                         ▼
┌─ FFT Thread (共用) ──────────────────────────────────────┐
│  FftProcessor_Serial (consume m_serMagBuffer)              │
│    → snapshot最新250点 → Hanning窗 → KissFFT               │
│    → bin=50 → atan2 → φ_Serial                            │
│    ↓ emit phaseReady(φ_Serial, ts)                        │
└──────────────────────────────────────────────────────────┘
                         │
                         ▼
┌─ UI Thread ──────────────────────────────────────────────┐
│  PhaseBuffer 追加 φ_Serial                                │
│  Δφ = φ_X1 − φ_Serial → DiffBuffer                        │
└──────────────────────────────────────────────────────────┘
```

```
│ TCP 通信线程  │  │串口 通信线程  │  │ TCP 解析线程  │  │串口 解析线程  │  │   UI 线程    │
│ (QThread)    │  │ (QThread)    │  │ (QThread)    │  │ (QThread)    │  │  (主线程)     │
├──────────────┤  ├──────────────┤  ├──────────────┤  ├──────────────┤  ├──────────────┤
│ ChanMgr_TCP  │  │ ChanMgr_Ser  │  │ MagDecoder   │  │ DollarParser │  │ MagMainWindow│
│ TcpChannel   │  │ SerialChannel│  │ MagFormula   │  │ MagFormula   │  │ RealTimeChart│
│              │  │              │  │ MagBuf_TCP   │  │ MagBuf_Ser   │  │ ×2           │
└──────┬───────┘  └──────┬───────┘  └──────┬───────┘  └──────┬───────┘  └──────┬───────┘
       │                 │                 │                 │                 │
       │  readyRead      │  readyRead      │                 │                 │
       ├────────────────►│                 │                 │                 │
       │                 ├────────────────►│                 │                 │
       │                 │                 │  magReady       │                 │
       │                 │                 ├────────────────►│                 │
       │                 │                 │                 │  phaseReady     │
       │                 │                 │                 ├────────────────►│
       │                 │                 │                 │                 │
       └─────────────────┴─────────────────┴─────────────────┴─────────────────┘
                     全部跨线程使用 Qt::QueuedConnection

┌──────────────┐
│  FFT 线程     │    ← 共用同一线程，跑 3 个 FftProcessor 实例
│ (QThread)    │       单次 FFT < 1ms，3 实例并行 < 3ms，50Hz 远未饱和
├──────────────┤
│ FftProc_X1   │
│ FftProc_Y1   │
│ FftProc_Ser  │
└──────────────┘
```

**线程数**：5 个（TCP 通信 / 串口通信 / TCP 解析 / 串口解析 / FFT）+ 主线程 = **共 6 线程**。

## 4. 模块接口定义

### 4.1 MagCollectorFrame

```cpp
// include/MagCollectorFrame.h
struct MagCollectorFrame {
    static constexpr uint8_t  HEADER_H    = 0x46;   // 'F'
    static constexpr uint8_t  HEADER_L    = 0x4D;   // 'M'
    static constexpr uint8_t  FOOTER_H    = 0x0D;   // CR
    static constexpr uint8_t  FOOTER_L    = 0x0A;   // LF
    static constexpr int      CHANNEL_COUNT = 6;
    static constexpr int      RAW_FRAME_SIZE = 48;

    uint16_t  frameLength;    // 帧总长（当前恒为48）
    uint32_t  sequence;       // 帧序列号
    uint8_t   timestamp[12];  // WW/MM/DD/YY/HH/MM/SS/FF/MS(4B)
    int32_t   data[6];        // 6通道 int24 → int32 (符号扩展)
    uint8_t   flags[6];       // 通道标识 'X','Y','Z','x','y','z'
    uint16_t  checksum;       // 接收到的校验和
};
```

## 5. 关键技术决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| FFT 库 | KissFFT（实数 FFT） | MIT 协议，单文件，1000 点 < 0.5ms |
| TCP 窗口 | N=1000 | 1000Hz/1000点 → 1Hz 分辨率, 50Hz 在 bin 50 精确对齐 |
| 串口窗口 | N=250 | 250Hz/250点 → 1Hz 分辨率, 50Hz 在 bin 50 精确对齐 |
| FFT 输出率 | 50 Hz | 与 RealTimeChart 40ms 定时器接近，满足 50Hz 实时性要求 |
| 窗函数 | Hanning | 平衡主瓣宽度与旁瓣衰减，工业常用 |
| DataBuffer 改造 | 模板化 `DataBuffer<T>` | 改动最小，复用锁/环形逻辑 |
| 双通道架构 | 完全独立管道 | TCP/串口互不阻塞，故障隔离 |
| 相位差计算 | UI 层计算 | 不耦合到 FftProcessor，方便扩展 |
| 帧长度字段 | 保留读取但当前恒为 48 | 兼容未来帧格式变更 |

## 6. FFT 参数数学验证

### 6.1 TCP (1000 Hz)

```
fs = 1000 Hz,  N = 1000,  Δf = fs/N = 1.0 Hz
f_target = 50 Hz  →  bin = f_target / Δf = 50  ✅ 精确整数

相位提取:
  Z = FFT{windowed_samples}[50] = R + j·I
  φ = atan2(I, R) × 180/π  ∈ [-180°, +180°]
```

### 6.2 串口 (250 Hz)

```
fs = 250 Hz,  N = 250,  Δf = fs/N = 1.0 Hz
f_target = 50 Hz  →  bin = f_target / Δf = 50  ✅ 精确整数

相位提取: 同上
```

### 6.3 为什么不用 N=512 或 N=1024

| N | TCP bin(50Hz) | 串口 bin(50Hz) | 频谱泄漏 |
|:--|:--|:--|:--|
| 512 | 25.6 | — | ❌ 严重 |
| 1000 | **50** | — | ✅ 无 |
| 1024 | 51.2 | — | ❌ |
| 250 | — | **50** | ✅ 无 |
| 256 | — | 51.2 | ❌ |

KissFFT 支持任意长度（不仅限于 2 的幂），故直接使用 N=1000 / N=250。
