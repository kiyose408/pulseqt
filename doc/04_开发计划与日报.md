## Day 2 — 数据管理 + SQLite + 实时曲线

**日期**：05-31（周日） | **预计工时**：8-10h

### 上午（4h）：数据层

- [ ] 定义 `DataPoint` 结构体（timestamp + QVector\<double\> channels）
- [ ] 实现 `DataBuffer` 环形缓冲（QVector + 头尾指针，QMutex 保护，默认 10000 条）
- [ ] 实现 `DataTableModel : QAbstractTableModel`
  - `rowCount()` / `columnCount()` / `data()` / `headerData()`
  - 自动滚到最新行（`scrollToBottom`）
  - `Qt::BackgroundRole`：阈值超限红色标记
- [ ] 实现 `DatabaseManager`
  - `initDB()`：建表 + `PRAGMA journal_mode=WAL`
  - `batchInsert()`：事务包裹批量写入（100 条/批）
  - `query(t_begin, t_end)`：时间范围查询
- [ ] 将 DataTableModel 挂载到 MainWindow 右侧 QTableView
- [ ] 验证：跑 simulator → 表格实时滚动 → `sqlite3 data.db` 确认数据落盘

### 下午（4h）：实时曲线

- [ ] 实现 `RealTimeChart : QWidget`
  - `paintEvent`：绘制坐标轴、网格、多通道曲线
  - `QPainterPath` 增量追加（只追加新点，不重建整个路径）

- [ ] 曲线交互：

- [ ] 多通道颜色配置（QColor 数组，8 种预设颜色）
- [ ] 将 RealTimeChart 集成到 MainWindow 左侧（QSplitter 7:3）
- [ ] 验证：100Hz 数据流 → 曲线 30+ FPS 流畅滚动

### 晚间（2h）：收尾

- [ ] 采集 30 分钟数据 → 确认曲线不卡顿、SQLite 文件大小合理

## Day 3 — 多线程架构 + 历史回放 + CSV 导出

**日期**：06-01（周一） | **预计工时**：8-10h

### 上午（4h）：多线程改造

- [ ] 实现 `SerialWorker : QObject`（封装 QSerialPort）
- [ ] 实现 `TcpWorker : QObject`（封装 QTcpSocket）
- [ ] 实现 `ParseWorker : QObject`（持有 ProtocolDecoder + DataBuffer + DatabaseManager）
- [ ] MainWindow 创建线程 + moveToThread：
  ```cpp
  m_commThread  = new QThread;   // 通信线程
  m_parseThread = new QThread;   // 解析线程
  m_serialWorker->moveToThread(m_commThread);
  m_parseWorker->moveToThread(m_parseThread);
  ```
- [ ] 信号槽全队列连接（`Qt::QueuedConnection`）：
  - `SerialWorker/TcpWorker::rawDataReceived → ParseWorker::onRawDataReceived`
  - `ParseWorker::dataPointReady → MainWindow::onDataPointReady`（UI 更新）
  - `ParseWorker::dbWriteRequest → DatabaseManager::batchInsert`
- [ ] 生命周期管理：窗口关闭 → 停止采集 → `quit()` → `wait()` → `deleteLater`
- [ ] 验证：`qDebug() << QThread::currentThread()` 确认各对象所在线程正确

### 下午（4h）：历史回放 + CSV 导出

- [ ] 实现 `HistoryPlayer`（底部控制条）
  - QSlider：时间轴（最早 → 最晚），QLabel 显示当前时间
  - Play/Pause 按钮 + 速度选择（1x/2x/5x/10x）
  - 拖动滑块 → `DatabaseManager::query()` → 更新 DataTableModel → 重绘曲线
- [ ] 实现 `ExportDialog`
  - QDateTimeEdit ×2 选时间范围
  - QCheckBox 列表选通道
  - 导出按钮 → QFileDialog 选路径 → 写 CSV
- [ ] 验证：采集 30min → 拖动滑块回放 → 导出 CSV → Excel 打开验证

### 晚间（2h）：收尾

- [ ] 启动 72h 压测（后台挂着跑 simulator + 上位机）
- [ ] `git commit -m "Day 3: 三线程架构 + 历史回放 + CSV 导出"`

### 验收标准

```
qDebug() 确认：通信线程 ≠ 解析线程 ≠ 主线程
采集运行 30min → UI 不卡顿，曲线流畅
拖动 HistoryPlayer 滑块 → 曲线跳到对应时刻，点播放 → 自动前进
导出 CSV → Excel 打开 → 时间戳 + 通道数据完整正确

```

## 风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|:--:|:--:|------|
| 串口硬件不可用 | 中 | 高 | 使用 socat 虚拟串口对，开发完全不受影响 |
| 自定义曲线性能不达标 | 中 | 高 | 降级为 QChart 方案（QLineSeries），放弃自绘 |
| 多线程出现死锁 | 低 | 高 | 所有数据传递用队列信号槽，不用共享内存+锁，从设计上杜绝死锁 |
| 时间不够 | 中 | 中 | Day 3 多线程降级为两线程（通信线程 + 主线程），回放砍掉只保留 CSV 导出 |
