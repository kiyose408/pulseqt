## 每个 T 任务的标准流程

1. **分配** — Tech Lead 在 `开发日志记录.md` 中填写任务详细 + 验收标准 + 开发建议
2. **编码** — 开发者按建议路线实现
3. **自测** — 对照验收标准逐条打勾
4. **验收** — Tech Lead 审查代码 + 确认标准通过
5. **推送** — `git commit -m "Txxx: 标题" && git push origin dev`
6. **记录** — 在日志中补全 Bug 解决、知识点回顾、核心代码

#### T022 — ProtocolDecoder 单元测试

| 项目       | 内容                                                                                       |
| -------- | ---------------------------------------------------------------------------------------- |
| **目标**   | 用 QtTest 覆盖状态机全部 7 个状态，含正常帧 + 粘包/半帧/CRC 错误/乱码注入                                              |
| **测试用例** |                                                                                          |
| ①        | 完整帧解码：`feed(完整帧)` → `QVERIFY(frameDecoded 信号触发)` → 校验 Frame 各字段                         |
| ②        | 粘包场景：两个完整帧拼在一起 `feed()` → 验证两次 `frameDecoded`                                           |
| ③        | 半帧续传：feed 帧前半段 → 验证无信号 → feed 后半段 → 验证 `frameDecoded`                                    |
| ④        | CRC 错误：篡改 CRC 字节 → 验证 `crcError` 信号触发 → 状态机回到 `WAIT_HEADER_H`                            |
| ⑤        | 帧头误识别：Payload 中含 `0xA5 0x5A` 不误判 → 按长度字段正确跳过                                       |
| ⑥        | 乱码注入：随机字节流 → 状态机不崩溃、不内存泄漏 → 最终回到 `WAIT_HEADER_H`                                       |
| ⑦        | `reset()` 测试：解码中途 reset → 状态归零 → 后续帧正常解码                                                |
| ⑧        | CRC16 标准向量：`"123456789"` → `0x29B1`（在 Frame 单元测试中）                                      |
| **文件**   | `tests/TestProtocolDecoder.cpp`（新增）, `tests/TestFrame.cpp`（新增）                            |
| **框架**   | QtTest（`QTEST_MAIN`），无需 mock，`ProtocolDecoder::feed()` 输入 `QByteArray`，输出 signal，天然可测 |
| **验收标准** | `ctest -R ProtocolDecoder` 全部 pass，`gcov` 行覆盖率 ≥ 95%                                     |

#### T029 — IFilter 抽象接口 + FilterPipeline

| 项目       | 内容                                                                                       |
| -------- | ---------------------------------------------------------------------------------------- |
| **目标**   | 定义数据处理管道，用策略模式实现可插拔的信号处理链                                                                   |
| **设计**   |                                                                                          |
| ①        | `IFilter` 抽象接口：`QVector<double> process(const QVector<double> &input)` + `void reset()`           |
| ②        | `FilterPipeline`：持有 `QVector<IFilter*>`，对每个 DataPoint 依次执行 `process()`                           |
| ③        | Pipeline 插入 ParseWorker：`dataPointReady → FilterPipeline → DataBuffer`                            |
