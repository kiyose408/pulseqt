## 1. TCP 输入 — 六通道磁场集采器

### 1.1 物理层

| 参数 | 值 |
|:--|:--|
| 传输方式 | TCP 客户端（上位机主动连接下位机） |
| 采样率 | **1000 Hz** |
| 字节序 | **小端序 (Little-Endian)** |

### 1.2 帧格式（固定 48 字节）

```
Offset  Size  Field                  Value / 说明
──────────────────────────────────────────────────────
 0      2     帧头                   0x46 0x4D  "FM" (ASCII)
 2      2     帧总长                 uint16 LE = 48
 4      4     帧序列号               uint32 LE, 递增
 8      1     周 WW                  
 9      1     月 MM                  
10      1     日 DD                  
11      1     年 YY                 0~99
12      1     时 HH                  
13      1     分 MM                  
14      1     秒 SS                  
15      1     时间格式 FF            
16      4     毫秒 MS               uint32 LE
──────────────────────────────────────────────────────
20      3     X1 通道数据            int24 LE (有符号, 3 字节)
23      1     X1 通道标志            0x58 'X'
24      3     Y1 通道数据            int24 LE
27      1     Y1 通道标志            0x59 'Y'
28      3     Z1 通道数据            int24 LE (不使用)
31      1     Z1 通道标志            0x5A 'Z'
32      3     X2 通道数据            int24 LE (不使用)
35      1     X2 通道标志            0x78 'x'
36      3     Y2 通道数据            int24 LE (不使用)
39      1     Y2 通道标志            0x79 'y'
40      3     Z2 通道数据            int24 LE (不使用)
43      1     Z2 通道标志            0x7A 'z'
──────────────────────────────────────────────────────
44      2     校验和                 uint16 LE
            校验范围: offset 4~43 共 40 字节的简单字节累加
            (序列 4B + 时间 12B + 通道数据 24B)
46      2     帧尾                   0x0D 0x0A  CR LF
──────────────────────────────────────────────────────
```

### 1.3 int24 解码

```cpp
int32_t decodeInt24(const uint8_t bytes[3]) {
    uint32_t raw = bytes[0] | (bytes[1] << 8) | (bytes[2] << 16);
    if (raw & 0x800000)  // bit23 = 1 → 负数
        return (int32_t)(raw | 0xFF000000);
    return (int32_t)raw;
}
// 范围: [-8,388,608, +8,388,607]
```

### 1.4 校验规则

1. **校验和** = sum(offset 4~43 共 40 字节) & 0xFFFF
2. **帧尾** = 0x0D 0x0A
3. **通道标志** = X(0x58) Y(0x59) Z(0x5A) x(0x78) y(0x79) z(0x7A) — 顺序和值必须严格匹配

三项全部通过才视为有效帧。

### 1.5 使用通道

**仅 X1 (data[0]) 和 Y1 (data[1])** 参与后续计算，其余 4 通道忽略。

## 2. 串口输入 — $ 格式磁传感器

### 2.1 物理层

| 参数 | 值 |
|:--|:--|
| 传输方式 | 串口 (UART) 或 TCP 透传 |
| 采样率 | **250 Hz** |
| 编码 | ASCII 文本 |

### 2.2 数据格式

```
$xxxxxxx.xxxxxx\n

正则: ^\$(\d{7})\.(\d{6})$

示例: $1234567.123456\n
      $0000042.000001\n
```

- `$` 开头
- 7 位整数 (0~9,999,999)，不足 7 位左侧补 0
- `.` 小数点
- 6 位小数 (0~999,999)
- `\n` 或 `\r\n` 结尾

### 2.3 解析规则

```
1. 按 \n 拆行
2. 跳过空行
3. 跳过不以 '$' 开头的行
4. 匹配正则: ^\$(\d{7})\.(\d{6})$
5. value = 整数部分 + 小数部分 / 1,000,000
6. 格式不匹配 → 丢弃 + 日志
7. 半行（末尾无 \n）→ 缓存到下次
```

## 3. 磁场值计算

### 3.1 TCP — ADC → 磁场值

```
Mag = raw_ADC × 100,000 / 8,388,607

其中:
  raw_ADC : int24 有符号值, 范围 ±8,388,607
  100,000  : 2.5V × 1000 × 40 的合成系数
  8,388,607: 2^23 − 1 (23-bit ADC 满量程)
```

**X1 和 Y1 各自独立计算** Mag_X1 和 Mag_Y1，不合成矢量。

```cpp
constexpr double ADC_MAX = 8388607.0;
constexpr double SCALE   = 100000.0;

double mag_tcp(int32_t raw) {
    return (double)raw * SCALE / ADC_MAX;
}
```

### 3.2 串口 — Value → 磁场值

```
Mag = parsed_value / 3.5
```

## 4. 时间戳规范

**全部使用上位机时间**，不依赖下位机时间戳。

```
Timestamp = QDateTime::currentMSecsSinceEpoch()
```

打戳时机：
- TCP：帧解码完成时
- 串口：$ 行解析完成时

## 5. FFT 相位提取

### 5.1 参数

| 参数 | TCP | 串口 | 公式 |
|:--|:--|:--|:--|
| 采样率 fs | 1000 Hz | 250 Hz | 固定 |
| FFT 点数 N | **1000** | **250** | N = fs → Δf = 1.0 Hz |
| 频率分辨率 Δf | 1.0 Hz | 1.0 Hz | fs / N |
| 目标频率 f₀ | 50 Hz | 50 Hz | 工频 |
| 目标 bin | **50** | **50** | f₀ / Δf |
| 窗函数 | Hanning | Hanning | w[n] = 0.5×(1−cos(2πn/(N−1))) |
| FFT 库 | KissFFT (实数) | KissFFT (实数) | BSD-3-Clause |
| 输出率 | 50 Hz | 50 Hz | QTimer 20ms |

### 5.2 为什么不用 N=1024

| N | Δf | 50Hz bin | 频谱泄漏 |
|:--|:--|:--|:--|
| 512 | 1.953 | 25.6 | ❌ |
| 1000 | **1.0** | **50** | ✅ |
| 1024 | 0.977 | 51.2 | ❌ |
| 2000 | 0.5 | 100 | ✅ |

### 5.3 相位计算公式

```
bin = 50

Z = FFT{windowed_samples}[bin] = R + j·I

φ = atan2(I, R) × 180 / π    范围 [−180°, +180°]
```

## 6. 相位差计算

```
Δφ_X1 = normalize(φ_X1 − φ_Serial)
Δφ_Y1 = normalize(φ_Y1 − φ_Serial)

normalize(d): while d>180 d-=360; while d<-180 d+=360; return d;
```

仅当 TCP(X1) 和串口都有有效数据时才计算相位差。

## 7. 数据流

```
┌─────────────────┐     ┌─────────────────┐
│ TCP 1000Hz      │     │ 串口 250Hz       │
│ "FM"...0D0A帧   │     │ $xxxx.xxxxxx\n   │
└────────┬────────┘     └────────┬────────┘
         │ 帧解码                │ $解析
         ▼                       ▼
    raw_X1, raw_Y1          parsed_value
         │                       │
         ▼ MagFormula            ▼ MagFormula
    Mag_X1, Mag_Y1           Mag_Serial
         │                       │
         ▼ FFT(N=1000)           ▼ FFT(N=250)
      φ_X1, φ_Y1              φ_Serial
         │                       │
         └───────────┬───────────┘
                     ▼
              Δφ_X1, Δφ_Y1
                     │
                     ▼
              波形展示 (X=时间, Y=角度)
```

## 9. 单元测试向量

### 9.1 帧解码

| 输入 | 预期 |
|:--|:--|
| 完整 48B 帧, seq=42 | 解析成功, sequence=42 |
| 帧头前有杂字节 0x00 0x01 0x02 0x03 | 跳过杂字节, 正确解析帧 |
| 'F'+0x00 (假帧头) | 丢弃, 在后续数据中重新找 "FM" |
| 帧尾 0x0D 0x0A 改为 0x00 0x00 | 校验失败, 不输出 |
| 篡改校验和 | checksumError |
| 2 帧拼接 | 2 帧均正确解析 |
| 帧分两次 feed (前半/后半) | 正确拆包 |
| X1 标志改为 0x00 | 标志校验失败 |
| 1000 帧连续 | 1000 帧全部正确 |

### 9.2 $ 格式解析

| 输入 | 预期 |
|:--|:--|
| `$1234567.123456\n` | value=1234567.123456 |
| `$0000042.000001\n` | value=42.000001 |
| `$0000000.000000\n` | value=0.0 |
| `$9999999.999999\n` | value=9999999.999999 |
| 3 行拼接 | 3 次 valueReady |
| 半行 `$123...` 无 \n | 缓存, 等待后续数据 |
| `garbage\n$0000001.000000\n` | 跳过 garbage, 解析 $ 行 |
| `$abc\n` | parseError, 不输出 |
| `\r\n` 结尾 | 正确解析 |

### 9.3 磁场计算

| 输入 | 公式 | 预期 |
|:--|:--|:--|
| raw=0 | fromAdc | 0.0 |
| raw=8,388,607 | fromAdc | 100,000.0 ± 0.001 |
| raw=−8,388,607 | fromAdc | −100,000.0 ± 0.001 |
| value=3.5 | fromSerial | 1.0 |
| value=0.0 | fromSerial | 0.0 |

### 9.4 FFT 相位

| 输入信号 | fs / N | 预期 φ |
|:--|:--|:--|
| sin(2π·50·t + 0°) | 1000/1000 | 0° ± 2° |
| sin(2π·50·t + 45°) | 1000/1000 | 45° ± 2° |
| sin(2π·50·t − 90°) | 1000/1000 | −90° ± 2° |
| sin(2π·50·t + 30°) | 250/250 | 30° ± 2° |
| 不足 N/2 个样本 | — | 不输出, 不崩溃 |

## 10. 线程模型

```
通信线程(TCP)  →  解析线程(TCP)  →  MagBuffer_TCP
通信线程(串口) →  解析线程(串口) →  MagBuffer_Ser
                                      │
                                 FFT 线程 (共用)
                                  ↓ φ_X1, φ_Y1, φ_Serial
                                 UI 线程
                                  ↓ Δφ 计算 + 波形渲染
```

- 跨线程通信：`Qt::QueuedConnection`
- 共享缓冲区：`QMutex` 保护
- FFT 定时器：20ms, 必须在本线程启停
