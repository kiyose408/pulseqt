## 1. 分层架构

```
┌─────────────────────────────────────────────────┐
│                   UI 层 (ui/)                     │
│  MainWindow  StatusBar  RealTimeChart            │
│  DataTableView  HistoryPlayer  ExportDialog      │
├─────────────────────────────────────────────────┤
│                数据管理层 (data/)                  │
│  DataTableModel  DataBuffer  DatabaseManager     │
├─────────────────────────────────────────────────┤
│                业务逻辑层 (protocol/)              │
│  ProtocolDecoder  ProtocolValidator              │
├─────────────────────────────────────────────────┤
│                通信层 (communication/)            │
│  IChannel  SerialChannel  TcpChannel             │
│  ChannelManager                                  │
├─────────────────────────────────────────────────┤
│              工作线程层 (worker/)                  │
│  IChannel / ChannelManager     ParseWorker       │
└─────────────────────────────────────────────────┘
```

**核心原则**：上层依赖下层，下层不感知上层。层间通过**信号槽**或**抽象接口**通信。

## 2. 模块职责

### 2.1 通信层 (`communication/`)

```
IChannel (抽象接口)
├── SerialChannel  (QSerialPort 封装)
└── TcpChannel     (QTcpSocket 封装，客户端模式)

ChannelManager
  - 持有 IChannel*，管理连接生命周期
  - 提供 connectToDevice() / disconnect() / reconnect()
  - 发射信号：connected / disconnected / readyRead / errorOccurred
```

```
ChannelRegistry (全局注册表 + 工厂)
  - registerChannel(ChannelDescriptor) — 静态注册（各通道 .cpp 静态初始化块自注册）
  - availableChannels() → QVector<ChannelDescriptor>
  - create(id, config, parent) → IChannel*
```

**关键设计**：

- `IChannel` 只暴露 `open/close/write/signal`，上层不关心串口还是 TCP
- `ChannelManager` 内置重连定时器，检测到断开后启动指数退避重连
- `ChannelRegistry` 实现**开闭原则**：新增通道类型（如蓝牙 BLE、UDP）只需新建 IChannel 实现类 + 调用 `registerChannel()`，UI 的 `ConnectionDialog` 由注册表元数据驱动渲染，无需改动
- `ConnectionDialog` 通过 `availableChannels()` 动态生成通道下拉框和配置面板，扩展通道时对话框零改动

### 2.2 协议层 (`protocol/`)

```
Frame (struct)
  - header:     uint16  (0xA55A)
  - length:     uint8   (payload 字节数)
  - type:       uint8   (0xE1=数据帧, 0xE2=心跳, 0xE3=应答, 0xE4=握手请求, 0xE5=握手应答, 0xFF=错误)
  - payload:    uint8[]
  - crc:        uint16  (CRC16-CCITT)

ProtocolDecoder (7 状态机)
  状态: WAIT_HEADER_H → WAIT_HEADER_L → WAIT_LENGTH → WAIT_TYPE
        → WAIT_PAYLOAD → WAIT_CRC_L → WAIT_CRC_H → 回到 WAIT_HEADER_H

crc16_ccitt() — 自由函数，CRC16-CCITT 查表法（多项式 0x1021）
  声明于 Frame.h，实现在 ProtocolDecoder.cpp
```

**解码状态机**：

```
                ┌───────────────┐
                │ WAIT_HEADER_H │ ← 逐字节扫描 0xA5
                └──────┬────────┘
                       │ 找到 0xA5
                       ▼
                ┌───────────────┐
                │ WAIT_HEADER_L │ ← 期望 0x5A（否则回 HEADER_H）
                └──────┬────────┘
                       │ 找到 0x5A
                       ▼
                ┌─────────────┐
                │ WAIT_LENGTH │ ← 读 1 字节 (0~255)
                └──────┬──────┘
                       ▼
                ┌─────────────┐
                │  WAIT_TYPE  │ ← 读 1 字节
                └──────┬──────┘
                       ▼
                ┌──────────────┐
                │ WAIT_PAYLOAD │ ← 读 length 字节（定长跳过，不扫描内容）
                └──────┬───────┘
                       ▼
                ┌─────────────┐
                │ WAIT_CRC_L  │ ← 读 CRC 低字节
                └──────┬──────┘
                       ▼
                ┌─────────────┐
                │ WAIT_CRC_H  │ ← 读 CRC 高字节 → 拼合 → 校验
                └──────┬──────┘
                       │ CRC 通过 → emit frameDecoded(frame)
                       │ CRC 失败 → emit crcError → 丢弃
                       ▼
                 回到 WAIT_HEADER_H
```

### 2.3 数据管理层 (`data/`)

```
DataPoint (struct)
  - timestamp:  uint64   (毫秒时间戳)
  - channels:   QVector<double>

DataBuffer (环形缓冲区，线程安全)
  - 底层：QVector<DataPoint> + 头尾指针
  - 写入：QMutex 保护
  - 读取：snapshot() 在锁内拷贝一份，返回独立副本（按值），调用方遍历时无需额外加锁
  - 信号：bufferUpdated(int count)

DataTableModel : QAbstractTableModel
  - 行：数据点
  - 列：时间戳 + 各通道值
  - 角色：Qt::DisplayRole / Qt::BackgroundRole(阈值告警红色)
  - 更新：收到 bufferUpdated 信号后标记脏，100ms 节流定时器触发 beginResetModel / endResetModel 全量刷新

DatabaseManager
  - 使用 QSqlDatabase + QSqlQuery
  - 初始化：CREATE TABLE IF NOT EXISTS + PRAGMA journal_mode=WAL
  - 批量写入：每 100 条一个事务
  - 查询：SELECT * FROM data_points WHERE timestamp BETWEEN ? AND ? ORDER BY timestamp
  - 清理：DELETE FROM data_points WHERE timestamp < ? (保留最近 7 天)
```

### 2.4 工作线程层 (`worker/` · `communication/`)

> **v1.1 架构变更**：不再使用独立的 Worker 类。
> 通道操作统一走 `IChannel` 多态接口 + `ChannelManager` 生命周期管理。
> 通信线程持有 ChannelManager，解析线程持有 ParseWorker。

```
通信线程 (QThread #1)
  ChannelManager : QObject
    - 持有 IChannel*（多态：TcpChannel / SerialChannel）
    - 连接/断开/指数退避重连（1s→2s→…→30s）
    - 信号转发：connected / disconnected / readyRead / errorOccurred

  IChannel（抽象接口，在通信线程内创建）
    ├── TcpChannel    — QTcpSocket 封装，异步 open()
    └── SerialChannel — QSerialPort 封装，同步 open()

解析线程 (QThread #2)
  ParseWorker : QObject
    - 持有 ProtocolDecoder（7 状态机解码）
    - 持有 DataBuffer（环形缓冲 + QMutex）
    - 持有 DatabaseManager（SQLite WAL + 批量事务）
    - 持有 QTimer（心跳检测 5s/30s 超时）
    - 持有通道配置（握手协商结果）
    - rawDataReceived → decoder.feed() → frameDecoded
        → 按帧类型路由（心跳/握手/数据）
        → 数据帧解析为 DataPoint → push DataBuffer + insert DB
        → emit dataPointReady()
```

### 2.5 UI 层 (`ui/`)

```
MainWindow
  ├── 菜单栏：文件(导出CSV/退出) | 视图(曲线/表格/暗色主题) | 帮助(关于)
  ├── 工具栏：连接按钮 | 断开按钮 | 开始采集 | 停止采集
  ├── 中央区域 (QSplitter 水平分割)
  │   ├── RealTimeChart (左侧 70%)
  │   └── DataTableView (右侧 30%)
  ├── 底部：HistoryPlayer (QSlider + 播放/暂停按钮)
  └── 状态栏：连接状态 | 帧数 | 帧率 | 运行时长

RealTimeChart : QWidget
  - paintEvent 自绘
  - 每帧 paintEvent 从 DataBuffer snapshot 重建 QPolygonF（非增量），通过 lower_bound 裁剪可见窗口 + 像素抽稀优化
  - 双缓冲：QPixmap 离屏绘制 → QPainter::drawPixmap
  - 交互：wheelEvent 缩放 | mousePressEvent/MoveEvent 拖拽

DataTableView : QWidget
  - QTableView + DataTableModel
  - scrollToBottom() 自动跟随最新数据
  - 右键菜单：复制单元格值

HistoryPlayer : QWidget
  - QSlider (时间轴) + QLabel (当前时间) + QPushButton (播放/暂停)
  - 拖动滑块 → 从 SQLite 查询对应时间的数据 → 更新 DataTableModel → 重绘曲线
```

## 3. 线程模型

```
┌──────────────┐   队列信号槽   ┌──────────────┐   队列信号槽   ┌──────────────┐
│  通信线程     │ ────────────→ │  解析线程     │ ────────────→ │  UI 线程      │
│ (QThread #1) │ readyRead    │ (QThread #2) │ dataPointReady│ (主线程)       │
│              │              │              │               │              │
│ ChannelMgr   │              │ ParseWorker  │               │ MainWindow   │
│ └─IChannel   │              │ ProtocolDec. │               │ DataTableMod │
│  (Tcp/Serial)│              │ DataBuffer   │               │ RealTimeChart│
│              │              │ DBManager    │               │              │
└──────────────┘              └──────────────┘               └──────────────┘
```

**关键决策**：

- 通信和解析分两个线程：防止协议解析阻塞数据接收
- SQLite 写入放在解析线程：避免 UI 线程阻塞，且写入不需要单独线程（SQLite 串行写）
- 所有跨线程通信全部使用**队列信号槽**（`Qt::QueuedConnection`），参数自动深拷贝
- DataBuffer 环形缓冲由互斥锁保护，允许解析线程写入 + UI 线程读取

## 5. 数据流（一次完整采集）

```
1. 设备 → 串口/TCP 发送二进制帧
2. TcpChannel::QTcpSocket::readyRead / SerialChannel::QSerialPort::readyRead
3. → emit IChannel::readyRead(QByteArray)
4. → ChannelManager 信号转发 → emit readyRead(QByteArray)  (QueuedConnection)
5. → ParseWorker::onRawDataReceived() → decoder.feed(data)
6. ProtocolDecoder 状态机逐字节解析：
   WAIT_HEADER_H → WAIT_HEADER_L → WAIT_LENGTH → WAIT_TYPE
   → WAIT_PAYLOAD → WAIT_CRC_L → WAIT_CRC_H
7. CRC 校验通过 → emit frameDecoded(Frame)
8. ParseWorker lambda 按帧类型路由（心跳/握手/数据）
9. 数据帧 → parseDataPayload(frame.payload, dp)
    → DataBuffer::push(dp)  (QMutexLocker 保护)
    → DatabaseManager::insert(dp)  (缓冲至 100 条 → 批量事务 COMMIT)
    → emit dataPointReady()
10. UI 线程（QueuedConnection 接收）：
    a. DataTableModel → onBufferUpdated → 标记脏 → 100ms 节流 → beginResetModel
    b. RealTimeChart → QTimer 40ms → paintEvent → snapshot → computeYRange + drawCurves
```

## 6. 关键设计决策

| 决策      | 选择             | 理由                              |
| ------- | -------------- | ------------------------------- |
| 通信线程模型  | `moveToThread` | 职责分离清晰，QObject 不能跨线程移动但可以跨线程发信号 |
| 数据传递方式  | 队列信号槽          | 自动深拷贝参数，无需手动管理跨线程数据生命周期         |
| 曲线渲染方案  | QPainter 自绘    | QChart 在 100Hz 下性能不佳，自绘可控性更强    |
| 数据库写入策略 | 解析线程批量事务       | SQLite 串行写，批量事务比逐条快 10-50 倍     |
| 构建系统    | CMake          | 跨平台、Qt6 官方推荐、IDE 支持好            |
