# 05 — 磁场相位分析 · 需求规格说明

> **版本**：v1.0 | **日期**：2026-06-25

本文档定义 PulseQt v2.0 中"磁场相位分析"功能模块的需求、约束和验收标准。

---

## 1. 引言

### 1.1 背景

在 v1.1.1 通用多通道数据采集基础上，新增**双源磁场数据采集与 50Hz 相位分析**能力。通过同时采集 TCP（六通道磁场集采器）和串口（专用磁传感器）两路磁场数据，经 FFT 提取工频（50Hz）相位角，计算并展示相位差，为磁场比对和相位校准提供实时可视化手段。

### 1.2 术语

| 术语 | 说明 |
|------|------|
| **六通道磁场集采器** | 通过 TCP 发送 6 轴磁场原始数据的下位机设备，1000Hz 采样 |
| **串口磁传感器** | 通过串口发送 `$xxxxxxx.xxxxxx` ASCII 格式磁场值的独立传感器，250Hz 采样 |
| **Mag** | 磁场值（double），由原始 ADC 数据经物理公式换算得到 |
| **相位角 φ** | FFT 频谱中 50Hz 频点对应的相位，弧度转角度，范围 [-180°, +180°] |
| **相位差 Δφ** | φ_TCP − φ_Serial，归一化到 [-180°, +180°] |
| **滑动窗口 FFT** | 每次取最新 N 个样本做 FFT，窗口之间高度重叠，50Hz 输出 |

### 1.3 与 v1.x 的关系

- v1.x 的通信层（IChannel / TcpChannel / SerialChannel / ChannelManager）、数据层（DataBuffer）、日志系统全部复用
- 新增模块：MagCollectorDecoder、DollarParser、FftProcessor
- v2.0 与 v1.x 在同一项目中并存，通过 CMake 条件或独立可执行文件隔离（待定）

## 2. 功能需求

### 2.1 数据采集

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|:--:|
| F1.1 | TCP 六通道采集 | 连接磁场集采器 TCP 服务端，1000Hz 接收 48 字节帧 | P0 |
| F1.2 | 串口 `$` 格式采集 | 连接磁传感器串口，250Hz 接收 ASCII 数值行 | P0 |
| F1.3 | 双通道并行 | TCP 和串口同时连接、同时采集，互不阻塞 | P0 |
| F1.4 | 独立断线重连 | TCP / 串口各自独立重连，互不影响 | P1 |

### 2.2 协议解析

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|:--:|
| F2.1 | "FM" 帧解码 | 解析帧头 0x46 0x4D、帧长、序列、时间、6 通道 int24 数据、校验和、帧尾 0x0D 0x0A | P0 |
| F2.2 | 通道标志校验 | 验证 X1(0x58) Y1(0x59) Z1(0x5A) X2(0x78) Y2(0x79) Z2(0x7A) 标志位 | P0 |
| F2.3 | 校验和验证 | 序列+时间+通道数据共 40 字节的字节和校验，失败丢弃 | P0 |
| F2.4 | `$` 格式解析 | 解析 `$xxxxxxx.xxxxxx` 格式，处理粘包、半行、非法行 | P0 |
| F2.5 | 时间戳统一 | 所有数据以解析时刻上位机时间 `QDateTime::currentMSecsSinceEpoch()` 打戳 | P0 |

### 2.3 磁场计算

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|:--:|
| F3.1 | ADC→Mag (TCP) | Mag = raw × 100000.0 / 8388607.0（基于 2.5V 参考 / 23bit 分辨率 × 1000 × 40） | P0 |
| F3.2 | Serial→Mag | Mag = parsed_value / 3.0 | P0 |
| F3.3 | 通道独立计算 | X1 和 Y1 各自独立计算 Mag_X1、Mag_Y1 | P0 |

### 2.4 FFT 相位分析

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|:--:|
| F4.1 | 滑动窗口 FFT | TCP: N=1000 (1s), 串口: N=250 (1s), 均 50Hz 输出 | P0 |
| F4.2 | Hanning 窗 | 窗函数预处理，抑制频谱泄漏 | P0 |
| F4.3 | 50Hz 相位提取 | `atan2(im, re)` → 角度，bin 精确对齐（TCP bin=50, Serial bin=50） | P0 |
| F4.4 | 相位差计算 | Δφ_X1 = φ_X1 − φ_Serial, Δφ_Y1 = φ_Y1 − φ_Serial | P0 |
| F4.5 | 相位卷绕处理 | ±180° 跳变处 unwrap，保证波形连续（可选开关） | P2 |

### 2.5 可视化

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|:--:|
| F5.1 | 相位角波形 | RealTimeChart 展示 φ_X1、φ_Y1、φ_Serial 三条时间序列，X轴=时间，Y轴=角度 | P0 |
| F5.2 | 相位差波形 | RealTimeChart 展示 Δφ_X1-S、Δφ_Y1-S 两条时间序列 | P0 |
| F5.3 | Y 轴固定范围 | 相位角 [-180°, +180°]，相位差可配置（如 [-30°, +30°]） | P1 |
| F5.4 | 曲线交互 | 滚轮缩放、拖拽平移、右键重置（复用 RealTimeChart） | P1 |
| F5.5 | 原始磁场值曲线 | 可选：展示 Mag_X1 / Mag_Y1 / Mag_Serial 时间序列（时域波形） | P2 |

### 2.6 模拟器

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|:--:|
| F6.1 | TCP 帧模拟器 | Python 脚本，1000Hz，生成标准 48 字节帧，可注 50Hz 正弦波信号 | P0 |
| F6.2 | 串口模拟器 | Python 脚本，250Hz，生成 `$` 格式数值，可注 50Hz 正弦波信号 | P0 |
| F6.3 | 相位验证 | 模拟器中设已知初相，验证 FFT 输出相位与预设一致 | P0 |

## 3. 非功能需求

| 类别 | 指标 | 目标值 |
|------|------|:--:|
| **性能** | FFT 输出率 | 50 Hz（20ms 间隔） |
| **性能** | 单次 FFT 耗时 | < 1ms（1000 点 KissFFT） |
| **性能** | 曲线渲染帧率 | 25 FPS（RealTimeChart 定时器） |
| **性能** | 内存增量 | < 50MB（v1.1 基础 ~80MB → 总 < 130MB） |
| **可靠性** | 帧解码稳健性 | 1000Hz 下 1 小时无错帧、无漏帧 |
| **可靠性** | 串口解析稳健性 | 处理粘包、半行、格式错误行不崩溃 |
| **可用性** | 双通道同时连接 | 一次点击或两次点击完成，操作路径 < 3 步 |

---

## 4. 约束条件

| 约束 | 说明 |
|------|------|
| 复用 v1.1.1 基础 | IChannel / ChannelManager / DataBuffer / Logger / RealTimeChart 不改动或仅微小改动 |
| FFT 库 | KissFFT（MIT License，单文件 C，实数 FFT） |
| 采样率不一致 | TCP 1000Hz ≠ 串口 250Hz → FFT 窗口大小分别设计 |
| 50Hz 精确对齐 | bin = f × N / fs，N 必须能被 fs/gcd(fs,f) 整除 |
| 时间戳 | 统一使用上位机 `QDateTime::currentMSecsSinceEpoch()` |

---

## 5. 验收标准

| 编号 | 验收项 | 通过条件 |
|------|--------|----------|
| AC1 | 双通道采集 | 同时连接 TCP 模拟器 + 串口模拟器，两个 DataBuffer 均正常增长 |
| AC2 | 帧解码 | 模拟器发 10000 帧，解码器零错帧 |
| AC3 | 磁场计算 | 已知 ADC 值 → Mag 公式输出与预期一致（误差 < 1e-6） |
| AC4 | FFT 相位 | 模拟器灌 50Hz 正弦波（已知初相 45°），FFT 输出 45° ± 1° |
| AC5 | 相位差 | 两个信号初相差 10°，Δφ 显示 10° ± 1° |
| AC6 | 波形展示 | RealTimeChart 相位曲线正常滚动，缩放拖拽正常 |
| AC7 | 双通道独立性 | 断开 TCP 不影响串口采集，断开串口不影响 TCP |
| AC8 | 50Hz 长期稳定 | 连续运行 1 小时，无内存泄漏，相位值稳定不漂移 |

## 6. 模拟测试方案

### 6.1 TCP 模拟器验证链路

```
Python 模拟器 (1000Hz, 48B帧) ──TCP──→ PulseQt v2.0
  每个通道注入 50Hz 正弦波:         → MagCollectorDecoder
  data(t) = A·sin(2π·50·t + φ₀)    → MagFormula
  已知 φ₀ = 45°                     → FftProcessor(N=1000)
                                     → φ 输出 ≈ 45° ✓
```

### 6.2 串口模拟器验证链路

```
Python 模拟器 (250Hz, $格式) ──TCP/Serial──→ PulseQt v2.0
  value(t) = baseline + A·sin(2π·50·t + φ₀)  → DollarParser
  已知 φ₀ = 35°                               → /3.0
                                               → FftProcessor(N=250)
                                               → φ 输出 ≈ 35° ✓
```

### 6.3 相位差验证链路

```
TCP 模拟器   φ₀ = 45°
串口模拟器   φ₀ = 35°
           → Δφ 显示 ≈ 10° ✓
```
